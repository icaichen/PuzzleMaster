# ORG-LOG-001 带真假计数的唯一排位

状态：`PROTOTYPE`  
运行时内核：`CSP_ASSIGNMENT`

## 机制结构

- 每个物件是一个变量。
- 每个变量的域是可用展台位置。
- `ALL_DIFFERENT` 保证一台一物。
- 普通约束描述紧邻、不相邻、先后和固定偏移。
- `EXACTLY` 可以嵌套其他约束，表达“恰有N条陈述为真”。

## 内核当前支持的约束

- `ALL_DIFFERENT`
- `OFFSET`
- `LESS_THAN`
- `ADJACENT`
- `NOT_ADJACENT`
- `AT_END`
- `IN_SET`
- `EXACTLY`

所有约束使用 `TRUE / FALSE / UNKNOWN` 三值求值。部分赋值只有在确定违反规则时才被剪枝，避免把“尚未知道”错误地当成“不成立”。

## 求解

- 使用MRV（最少剩余值）选择下一个变量。
- 每次试值后立即进行约束传播式剪枝。
- 返回完整赋值解、访问节点数和剪枝统计。
- 解的数量按最终赋值计算，不把玩家不同的拖放顺序误算成不同答案。

## 线索必要性

样例的四条玩家可见规则逐条参与唯一性：

| 删除内容 | 剩余答案数 |
|---|---:|
| 钥匙紧靠天球仪右边 | 10 |
| 灯不与面具相邻 | 3 |
| 面具在灯左边 | 4 |
| 三条受损记录恰有一条错误 | 6 |

这项检查防止加入看似丰富但实际不影响答案的装饰线索。

## 表现层边界

未来界面可以拖动物件到展台，但约束内核只接收 `PLACE(variable, value)`。拖动距离、吸附动画和物件图片不参与答案判定。
