# ORG-ASM-001 单环线路拼板

## 机制目的

玩家旋转并放置不规则拼片，使其完整覆盖目标轮廓；同时，印在各拼片上的局部线路必须跨越拼片边界，组成一条没有断口、分支或多个独立小环的全局单环。

核心不是单纯试形状，也不是把管件放进固定格子：每块线路与其不规则外形绑定，旋转会同时改变占用形状和端口方向。

## 内核职责

- `PIECE_ASSEMBLY`：离散拼片的旋转、平移、轮廓覆盖、重叠与越界判定。
- 端口模块：相邻端口必须同时存在或同时不存在。
- 拓扑模块：全部占用格的线路图必须连通，且每个节点度数为 2，因此构成单环。
- `PACKING_GEOMETRY` 不在此机制中使用；它保留给连续坐标中的任意位置和碰撞装填。

## 状态与动作

- 状态：已经物化的拼片位姿列表。
- 动作：放置、移除、旋转后放置、重置。
- 固定拼片：由题面声明，运行时初始化时物化，不能移除或移动。

## 生成与质量门槛

1. 生成目标轮廓、目标单环和一个完整拼片分割。
2. 把目标单环的端口投影回每块拼片的局部坐标。
3. 枚举每块拼片的去重旋转和所有合法平移。
4. 使用最少剩余候选优先搜索完整装配。
5. 完整规则必须唯一解。
6. 去除线路规则后必须出现多种轮廓拼法，证明线路不是装饰。
7. 去除固定拼片后若出现整体对称解，必须保留真实的题面固定线索，不能由求解器暗中合并。
8. App 题还需验证操作手感、辨识度和推理节奏；机器唯一解只代表结构合格。

## 产品分层

- App：使用人工策划标准约束生成器，优先保留有明确局部突破口、较少无效试错的实例。
- 微信小游戏：可批量更换轮廓、分割和单环，但每题仍执行唯一性、线路必要性和对称性检查。
