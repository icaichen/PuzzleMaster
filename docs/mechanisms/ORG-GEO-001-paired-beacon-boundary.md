# ORG-GEO-001 成对信标边界

## 机制目的

玩家依次连接全部锚点，形成不自交的简单闭环。闭环必须同时满足精确面积和“每对信标恰有一个在内部”的异或条件。

这不是换皮的“把某物圈起来”：候选闭环首先由离散的锚点顺序产生，再由连续几何量和成对区域关系共同筛选。面积相同不代表信标关系相同，信标关系相同也不代表面积正确。

## 状态与动作

- 状态：锚点顺序 `order`、是否闭合 `closed`。
- 动作：加入锚点、撤回末端、闭合、重置。
- 非法动作：重复锚点、未知锚点、未使用全部锚点就闭合、形成自交线段。

## 判定

- 所有锚点恰好使用一次。
- 多边形简单且面积非零。
- 鞋带公式计算的双倍面积等于 `targetArea2`，整数坐标下无浮点面积误差。
- 每对信标恰有一点严格位于内部；落在边界上不算满足。

## 生成与质量门槛

1. 生成锚点和一条目标简单闭环。
2. 从目标闭环导出面积，再搜索信标对。
3. 穷举去除旋转与反向重复后的闭环。
4. 要求完整规则唯一解。
5. 分别移除面积规则、每一组信标规则；每次都必须产生多解，否则该规则冗余。
6. App 版本还需人工产品验收视觉可读性和推理节奏；机器唯一解不等于已经好玩。

## 产品分层

- App：锚点布局、面积数值、信标位置均经过生成器搜索和产品验收，限制视觉拥挤与试错量。
- 微信小游戏：沿用同一验证器，可扩大坐标与信标组合批量生成，但仍必须通过唯一解和规则必要性检查。
