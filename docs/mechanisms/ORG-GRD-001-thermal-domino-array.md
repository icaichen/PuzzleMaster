# ORG-GRD-001 冷热阵列

## 机制

玩家在固定网格候选位置放置带方向属性的骨牌。每块占两个相邻格，一端热、一端冷；全部格必须精确覆盖，热端构成正交不相邻集合，同时满足竖放骨牌的全局数量。

## 与 PIECE_ASSEMBLY 的边界

- 本机制的合法位置来自有限网格候选集合，求解核心是精确覆盖、局部禁配与聚合计数。
- `PIECE_ASSEMBLY` 处理拼片自身形状的旋转和平移，以及拼合后的边界或拓扑。
- 两者可以共用界面交互，但不能共用一个含糊的生成模板。

## 生成门槛

1. 枚举所有相邻格骨牌及两种冷热方向。
2. 选择固定候选、竖放目标和目标解。
3. 穷举精确覆盖，完整规则必须唯一解。
4. 移除热端禁邻规则必须多解。
5. 移除竖放计数规则必须多解。
6. 冗余的行列提示不得进入题面。
7. App 版本仍需产品验收推理节奏；微信版本可按相同门槛批量生成。
