# PuzzleMaster 原创谜题设计包 v1

## 目标

这是一个纯谜题合集，不包含故事主线。目标是做到：规则容易理解、交互本身参与推理、答案可验证、解出时有明确的“原来如此”，同时不复制任何现有游戏的题面、布局、数值、提示或解法。

第一版先验证三件事：

1. 能否从“核心洞察”而不是题材换皮出发设计谜题。
2. 能否把创意写成无歧义、可实现、可测试的规则。
3. 能否用程序排除意外解，而不是只验证设计者预想的答案。

## 原创性与质量门槛

每道题进入制作前必须通过以下检查：

- **去皮检查**：删除名称、美术和背景后，是否仍与某道参考题具有相同状态、操作、约束和解法结构？如果是，淘汰。
- **洞察检查**：能否用一句话描述玩家最终发现的关键关系？不能则说明题目只有繁琐操作。
- **必要性检查**：每条规则是否真正参与解题？删掉仍不影响答案的规则应删除。
- **公平性检查**：解题所需信息必须在题面或交互反馈中出现。
- **解空间检查**：程序枚举全部合法状态或动作序列，确认答案数量符合设计。
- **提示检查**：提示依次解释观察方向、关键关系和局部步骤，不能第一条就泄露答案。
- **试玩检查**：首次游玩者能复述规则；失败时知道自己违反了什么；成功主要来自推理而非穷举。

## 01 回声搬运工

### 核心体验

玩家不是分别控制两个人，而是在编写一段会被延迟重放的动作。关键洞察是：为了让搬运工到达自己的终点，某些看似绕路的动作其实是在为两回合后的回声铺路。

### 玩家可见规则

1. 棋盘为 4×4；深色格不可进入。
2. 每回合选择上、下、左、右之一，搬运工立即移动一格。
3. 回声会执行搬运工**两回合前**的指令；前两回合回声不动。
4. 任一角色撞墙、越界或两者停在同一格，本次方案立即失败。
5. 电梯将在第 7 回合结束时关闭。此时两者必须分别站在自己的出口。

### 初始布局

坐标从左上角 `(1,1)` 开始。

```text
┌───┬───┬───┬───┐
│ · │ E │ C │ · │
├───┼───┼───┼───┤
│ · │ · │ · │ · │
├───┼───┼───┼───┤
│ · │ c │ █ │ █ │
├───┼───┼───┼───┤
│ · │ e │ · │ · │
└───┴───┴───┴───┘
```

- `C`：搬运工，起点 `(3,1)`。
- `E`：回声，起点 `(2,1)`。
- `c`：搬运工出口 `(2,3)`。
- `e`：回声出口 `(2,4)`。
- `█`：墙壁 `(3,3)`、`(4,3)`。

### 操作与反馈

- 点击方向按钮或滑动输入一步。
- 屏幕下方显示最近两步指令，最旧的一步会发光并传给回声。
- 每回合先显示搬运工移动，再以半透明轨迹显示回声重放。
- 支持撤销一步和完整重置；提交由第 7 回合自动触发。

### 唯一解

`下 → 左 → 下 → 下 → 右 → 左 → 上`

验证程序枚举 `4^7 = 16,384` 个指令序列；只有该序列合法且在第 7 回合同时到达两个出口。

### 三级提示

1. 不要只看搬运工下一步的位置；每个指令还会在两回合后再次产生影响。
2. 搬运工第二步向左，并不是为了接近自己的出口，而是为了让回声在第四回合离开上方走廊。
3. 前三步是“下、左、下”。之后要先把搬运工送到第四行，再利用最后两步修正两人的纵向位置。

### 难度扩展

- v2：加入可由任一角色触发一次的压力门。
- v3：延迟从固定两回合变成棋盘上可读的节拍序列 `2, 1, 3`。
- 不通过单纯扩大棋盘增加难度。

## 02 叠色修复师

### 核心体验

五块彩色玻璃单独看都不完整。玩家必须根据成品中每个窗口的颜色，推断重叠位置上“谁必须在谁上面”，再恢复唯一的层叠顺序。

### 玩家可见规则

1. 有五条同样大小的透明玻璃，每条包含七个窗口。
2. 玻璃只能调整上下层级，不能翻面或左右移动。
3. 从正面看，每个窗口只显示该位置最上方的非透明颜色。
4. 必须使用全部五条玻璃，使七个窗口与目标样板完全一致。

### 玻璃数据

`.` 表示透明；颜色代号仅用于规格和测试。

| 玻璃 | 从左到右七个窗口 |
|---|---|
| A | `A...C.A` |
| B | `.A....B` |
| C | `B.DA.A.` |
| D | `.AD.A.B` |
| E | `...D..A` |

颜色映射：`A=琥珀`、`B=蓝`、`C=青`、`D=紫`。

目标样板：`BADDCAB`

### 操作与反馈

- 五条玻璃以独立托盘展示；拖动后插入层叠架的任意层。
- 成品预览实时更新，但不会标红错误窗口，避免把谜题变成逐格试错。
- 点击某条玻璃时，可暂时高亮它对当前成品可见的部分。
- 玩家确认后统一判定。

### 唯一解

从底层到顶层：`D → A → C → E → B`

验证程序枚举全部 `5! = 120` 种层序，只有这一种能产生目标样板。

### 推理骨架

- 第 5 格目标为 `C`，只有 A 玻璃能提供该颜色，因此 A 必须位于 D 上方。
- 第 1 格目标为 `B`，说明 C 必须位于 A 上方。
- 第 4 格目标为 `D`，说明 E 必须位于 C 上方。
- 此时第 7 格中的 D 已被 A、C、E 压住，若要显示 `B`，只能把 B 放到 E 上方。
- 四个窗口共同形成 `D < A < C < E < B` 的完整偏序，因此只有一个总顺序。

### 三级提示

1. 先看第 5 个窗口：目标中的青色只有一块玻璃能够提供。
2. 再依次观察第 1 和第 4 个窗口，可以得到三条首尾相接的上下关系。
3. 先把 D 放到底层；A 在 D 上，C 在 A 上，E 在 C 上，最后放 B。

### 难度扩展

- v2：允许两块玻璃翻面，但在边缘加入方向标记，确保推理而非盲试。
- v3：两种半透明颜色叠加产生第三色，层序和混色规则同时参与推理。

## 03 潮汐灯塔

### 核心体验

玩家每回合只能校准一层灯环，但回合结束后三层会按不同速度自动旋转。玩家要预测未来，而不是让当前三个缺口立即对齐。

### 玩家可见规则

1. 灯塔由内、中、外三层四向灯环组成，每层各有一个缺口。
2. 每个节拍开始时，必须选择一层顺时针旋转 90°。
3. 手动旋转后，若三个缺口处于同一方向，灯塔立即向该方向发射一次光束。
4. 发射判定后进入潮汐阶段：内环不动，中环顺时针 90°，外环顺时针 180°。
5. 电池只有三次发射机会。船只仅在第 3、5、7 拍经过北方；必须刚好照亮这三次，其他节拍不能误发射。

### 初始状态

- 四个方向依次编号：北 `0`、东 `1`、南 `2`、西 `3`。
- 内环缺口：北 `0`。
- 中环缺口：南 `2`。
- 外环缺口：东 `1`。
- 自动潮汐速度：内 `+0`、中 `+1`、外 `+2`。

### 操作与反馈

- 点击某一灯环完成本拍校准。
- 环旋转动画结束后才进行对齐检测，再播放潮汐自动旋转。
- 时间轴明确标出第 3、5、7 拍的船只；已消耗电量始终可见。
- 可以在提交前回看已执行的环编号，但不能修改中间一步，只能撤销或重置。

### 唯一解

按回合选择：`外、外、外、中、中、中、中`

发射轨迹：`无、无、北、无、北、无、北`。

验证程序枚举 `3^7 = 2,187` 个选择序列；仅此序列满足完整发射轨迹。

### 三级提示

1. 不要以当前对齐为目标；先计算一次潮汐后中环和外环会到哪里。
2. 前三拍只调整外环，可以在第三拍第一次向北发光。
3. 第三拍后停止调整外环，剩余四拍全部调整中环。

### 难度扩展

- v2：船只在不同方向经过，目标不再固定为北。
- v3：一次手动旋转可以选择 90° 或 180°，但总校准角度有限。

## 技术实现边界

第一批原型采用 2D 浏览器实现，优先选择 Phaser；题面、提示、设置和完成记录使用 DOM，棋盘和动画使用 Canvas。每道谜题拆为四个部分：

- `definition`：不可变关卡数据。
- `simulation`：纯状态转移，不依赖画面对象。
- `validator`：答案与失败条件验证。
- `presentation`：拖动、点击、动画、音效和视觉反馈。

验证脚本只证明规格中的解空间结论，不证明题目一定有趣。趣味性、表达和难度仍必须经过真人试玩。

## 下一阶段

1. 对十个概念做相似性检索，淘汰结构过近的设计。
2. 将前三题制作成无美术低保真可玩原型。
3. 每题邀请至少 5 名首次玩家测试，记录理解时间、首次成功时间、提示使用和错误原因。
4. 只有至少两题达到预期“洞察时刻”，才扩展第二批题目。
