# PuzzleMaster 可玩性审核标准 v1

## 结论等级

任何题目不得仅凭唯一解标记为“高质量可玩”。审核结论分为四层：

1. `LOGIC_VERIFIED`：规则可执行、解数量正确、关键规则必要。
2. `INTERACTION_VERIFIED`：自动操作能从初始状态完成正确解；撤销、重置、固定件和错误提交反馈有效。
3. `VISUAL_REVIEWED`：桌面与手机画面不存在关键遮挡、溢出或不可辨识状态；视觉没有泄露答案。
4. `PLAYER_VALIDATED`：真实玩家数据达到机制预设的完成率、提示率、操作错误率和退出率范围。

前三层可以在正式美术之前完成；第四层不能由 AI 模拟替代。

## 低保真测试台必须提供

- 真实谜题定义与真实运行时，禁止另写演示用答案逻辑。
- 完整主要操作、撤销、重置、提交和分层提示。
- 初始状态、进行中、错误状态、成功状态的明确视觉差异。
- 桌面与手机响应式布局。
- 可访问名称和键盘可聚焦控件。
- 可重复的自动完成路径和错误路径。

## 自动审核项目

- 页面与谜题资源成功加载，无控制台错误。
- 第一可操作画面无需展开大段说明。
- 求解器答案可以通过页面操作完整重放。
- 非法操作产生具体反馈，不破坏状态。
- 提交未完成、规则违规和成功分别反馈。
- 关键状态指标与运行时一致。
- 375–390px 宽度无横向溢出；关键按钮不遮挡棋盘。
- `prefers-reduced-motion` 下非必要动画停止。

## 启发式可玩性审核

以下项目可以由规则分析和模拟玩家辅助判断，但不能冒充真人结论：

- 首个突破口是否可从题面推导。
- 是否存在大量等价但无信息增益的操作。
- 错误路线是否能在合理步数内暴露。
- 提示是否逐层推进而不是直接给答案。
- 视觉编码是否增加认知负担或意外泄题。

## 真人验证最低记录

- 首次完成率与完成时间分布。
- 每一步撤销、重置、无效操作和停留时间。
- 各级提示打开率。
- 提交错误类型。
- 中途退出位置。
- 玩家对规则理解与“发现感”的简短反馈。

没有真人数据时，最高只能标记为 `VISUAL_REVIEWED`，不能标记为 `PLAYER_VALIDATED`。
