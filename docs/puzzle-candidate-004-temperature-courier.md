# 候选谜题 004：温差邮差

状态：`MACHINE_VERIFIED`，等待产品试玩验收  
机制族：`NAV-03 资源交换路线`  
目标产品：主 App 候选；微信生成机制候选

## 给玩家的题目

七间温室通过玻璃通道相连。邮差必须先到 C 和 E 取得两封信，再把它们送到 G。

每间温室都有一个温度牌：

- 邮差只能穿过连接**温差恰好为 1°C**的两间温室。
- 每穿过一次通道，两端温室的温度牌会立刻交换。
- 邮差从 A 出发，最多只能走 7 段通道。

你能完成投递吗？

## 地图

| 温室 | 初始温度 | 内容 | 相邻温室 |
|---|---:|---|---|
| A | 2°C | 起点 | B、D、E |
| B | 3°C | — | A、C、F |
| C | 3°C | 信件① | B、D |
| D | 4°C | — | A、C、F |
| E | 1°C | 信件② | A |
| F | 3°C | — | B、D、G |
| G | 3°C | 终点 | F |

```text
       E①
       │
       A ─── B ─── C②
       │     │    ╱
       D ─── F   ╱
       │     │  ╱
       └─────┘ ╱
             G

补充边：C—D；图中交叉线不表示额外连接。
```

实际连接集合：`A-B, B-C, A-D, A-E, D-F, F-G, B-F, C-D`。

## 核心洞察

邮差移动的不只是自己，也在把“当前房间的温度”带到下一间房。收取 C 的信后返回 B，会让 B 保持 2°C，从而打开原本温差为 0、无法通行的 B—F；随后同一温度继续被带到 F，又打开 F—G。

## 解答

`A → E → A → B → C → B → F → G`

关键温度变化：

1. `A(2) → E(1)`，交换后邮差所在 E 为 2。
2. 返回 A 后恢复 A/E 的初始温度，同时已取得 E 的信。
3. `A(2) → B(3)` 后，B 变为 2。
4. 访问 C 并返回 B，B 仍为 2，同时取得 C 的信。
5. 此时才能从 `B(2)` 进入 `F(3)`。
6. 交换后 F 变为 2，于是可以进入 `G(3)`。

## 三级提示

1. E 只有一条通道。如果要取 E 的信，什么时候去最不浪费步数？
2. 初始时 B 和 F 都是 3°C，无法直接通行。你需要把一个相差 1°C 的温度带到 B。
3. 先走 `A → E → A → B`。之后先取得 C 的信，再从 B 前往 F。

## 机器验证结果

- 最短解长度：7步。
- 最短解数量：1。
- 深度10以内探索到的规范化状态：19。
- 唯一最短路径：`A,E,A,B,C,B,F,G`。
- 求解器同时检查位置、七个温度、已取得信件集合和步数。

## 为什么不是单纯换皮路径题

去掉温室题材后，机制仍然是：移动会交换边两端的通行属性，交换后的属性由角色沿路径继续携带；玩家规划的是“位置路径”和“属性运输”两件事。若删除交换规则，关键边 B—F 与 F—G 永远无法按当前顺序打开，题目不成立。

## 当前风险

- E 和 C 都是支路式取件，可能让前半段显得过于容易。
- 7步上限强化了唯一性，但需要试玩确认玩家是否把它理解为合理约束，而非人为卡答案。
- 该题已证明逻辑成立，尚未证明达到主 App 的85分质量门槛。
